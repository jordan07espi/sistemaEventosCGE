# 📅 Sistema de Gestión de Eventos

![Banner del Proyecto](https://i.imgur.com/gKj3aZz.png)

Un panel de control web completo y dinámico para la gestión de eventos y usuarios. Este sistema permite a los administradores llevar un control total sobre los eventos programados, gestionar una base de datos de usuarios y visualizar estadísticas clave a través de un dashboard interactivo.

## 🚀 Características Principales

Este sistema ofrece un conjunto de herramientas robustas para la administración de eventos y usuarios.

* **Gestión de Usuarios:**
    * ✅ **Login Seguro:** Sistema de autenticación para proteger el acceso al panel.
    * 🔑 **Control de Roles:** Diferenciación entre usuarios **Administradores** y **Asistentes**.
    * ➕ **CRUD de Usuarios:** Funcionalidad completa para Crear, Leer, Actualizar y Eliminar (CRUD) perfiles de usuario.

* **Gestión de Eventos:**
    * 🗓️ **Calendario Interactivo:** Visualización de eventos en una interfaz de calendario intuitiva gracias a **FullCalendar**.
    * 🖱️ **Operaciones Drag & Drop:** Modifica las fechas de los eventos arrastrándolos y soltándolos directamente en el calendario.
    * 🎨 **CRUD de Eventos:** Completa funcionalidad para añadir, modificar y eliminar eventos con formularios modales (pop-ups).

* **Dashboard y Visualización de Datos:**
    * 📊 **Gráficos Dinámicos:** Estadísticas visuales, como la cantidad de eventos por mes, generadas con **Chart.js**.
    * 📈 **Métricas Clave:** Tarjetas de acceso rápido que muestran el número total de usuarios y eventos registrados.
    * 📄 **Tablas con Paginación:** Listado de usuarios y eventos en tablas interactivas con funcionalidad de búsqueda y paginación.

* **Experiencia de Usuario:**
    * ⚡ **Interfaz Responsiva:** Diseño totalmente adaptable a dispositivos móviles, tablets y de escritorio, construido con **Bootstrap**.
    * 🔔 **Alertas y Notificaciones:** Uso de `SweetAlert2` para mostrar mensajes de confirmación y error amigables.

## 🛠️ Instalación y Puesta en Marcha

Sigue estos pasos para instalar y ejecutar el proyecto en tu entorno local:

1.  **Requisitos Previos:**
    * Entorno de desarrollo local como **XAMPP**, WAMP o MAMP.
    * Acceso a un gestor de bases de datos como **phpMyAdmin**.

2.  **Copiar los Archivos:**
    * Copia la carpeta `sistemaEventos` en el directorio `htdocs` de tu instalación de XAMPP.
        * Ubicación de ejemplo: `C:\xampp\htdocs\sistemaEventos`

3.  **Crear la Base de Datos:**
    * Abre **phpMyAdmin** desde tu panel de XAMPP.
    * Crea una nueva base de datos con el nombre `eventos_db`.
    * Importa el archivo `eventos_db.sql` que se encuentra dentro de la carpeta del proyecto. Este archivo creará las tablas `usuarios` y `eventos`, además de un usuario administrador por defecto.

4.  **Verificar la Conexión:**
    * Abre el archivo `config/db.php` y asegúrate de que las credenciales coincidan con las de tu entorno de MySQL. Por defecto, las credenciales para XAMPP son correctas.

5.  **Acceder al Sistema:**
    * Abre tu navegador y ve a `http://localhost/sistemaEventos/`
    * Inicia sesión con las credenciales del administrador por defecto:
        * **Usuario:** `admin`
        * **Contraseña:** `admin`
    * ¡Listo! Ya puedes empezar a gestionar tus eventos.

## 📂 Estructura del Proyecto

La estructura del proyecto está diseñada para ser intuitiva y escalable, separando la lógica, la presentación y los recursos.

```
/sistemaEventos
│
├── 📂 assets/
│   ├── 📂 css/         # Estilos CSS personalizados
│   ├── 📂 js/          # Lógica JavaScript (main.js, login.js, etc.)
│   └── 📂 vendor/      # Librerías de terceros (Bootstrap, FullCalendar, etc.)
│
├── 📂 config/
│   └── db.php          # Configuración de conexión a la base de datos
│
├── 📂 models/
│   ├── Eventos.php     # Lógica de negocio para eventos (CRUD)
│   └── Usuarios.php    # Lógica de negocio para usuarios (CRUD, login)
│
├── 📂 views/
│   ├── 📂 includes/
│   │   ├── footer.php
│   │   └── header.php
│   ├── calendario.php  # Vista del calendario de eventos
│   ├── chart.php       # Vista de los gráficos y estadísticas
│   ├── home.php        # Página principal del dashboard
│   └── usuarios.php    # Vista para la gestión de usuarios
│
├── controller/
│   ├── eventos.php     # Controlador para las acciones de eventos (AJAX)
│   └── usuarios.php    # Controlador para las acciones de usuarios
│
├── index.php           # Punto de entrada y gestión de login
└── logout.php          # Script para cerrar la sesión

```
